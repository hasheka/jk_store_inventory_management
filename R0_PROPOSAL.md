## The Problem

JK Store manages stock for 10–20 grocery items using a **hand-written register**. Every morning the owner spends 10–15 minutes checking stock levels manually. Despite this, items frequently run out before the next restock — causing lost sales. There is also no record of which items sell fastest, so restocking decisions are based entirely on memory and guesswork.

**Manual process today:**
Owner physically checks shelves → writes stock in register → decides what to order → calls supplier.

**Problems:**
- Items run out before anyone notices
- No data on which items sell most
- No history of past sales or restocks
- Wasted time on manual counting

---

## Proposed Solution

**JK Store Inventory Manager** — a web-based inventory management system that runs entirely in the browser (no server required).

**Features:**
1. Live dashboard with stock alerts (critical / low / healthy)
2. Full inventory table with search and category filters
3. Sale recording that auto-deducts from stock
4. Restock module for low-stock items
5. Activity log of all sales and restocks
6. Sales report with revenue and top sellers
